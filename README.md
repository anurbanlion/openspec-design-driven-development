# OpenSpec: Design-Driven Development 🎨🚀

This repository implements a modified OpenSpec workflow optimized for Design-Driven Development. Unlike the traditional specification-heavy approach, this simplified flow centers on design (`design.md`) as the single source of truth, minimizing specs overhead and allowing developer checklists (`tasks.md`) to be derived directly from the design artifacts.

The project includes configuration files under `openspec/` defining the `design-driven-development` schema and its templates, alongside project-specific conventions for component creation and integrations (ex. Figma, Storybook) structures. This setup ensures that implementation remains tightly aligned with the UI design and general guidelines from the very beginning.
